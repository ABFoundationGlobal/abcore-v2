# Parlia Genesis System Contract Bytecodes

This directory contains the compiled bytecode for all system contracts injected at `ParliaGenesisBlock`
(the block where a Clique chain migrates to Parlia consensus). Each file holds raw hex-encoded
`deployedBytecode` for one contract, embedded into the Go binary via `//go:embed`.

## How to Generate the Bytecodes

The bytecodes are generated by compiling [bsc-genesis-contract](https://github.com/bnb-chain/bsc-genesis-contract)
with chain-specific parameters. The Makefile automates the full pipeline.

### Prerequisites

- [Foundry](https://book.getfoundry.sh/getting-started/installation): `curl -L https://foundry.paradigm.xyz | bash && foundryup`
- Python 3.10 / 3.11 / 3.12 (`rpds-py` does not support Python 3.14+)
- Node.js >= 16
- `jq`

Poetry is installed automatically by `make pre` if not already present.

### Step 1: Prepare bsc-genesis-contract

```bash
make pre
```

This clones bsc-genesis-contract (v1.2.5) into this directory, installs Node.js and Python
dependencies, and installs the `forge-std` library.

### Step 2: Configure validators

Copy the example config and fill in your validator info:

```bash
cp validators.conf.example validators.conf
```

Each line in `validators.conf` is one validator (comma-separated, no spaces, no comment lines):

```
consensusAddr,feeAddr,bscFeeAddr,votingPower,bLSPublicKey
```

| Field | Description |
|-------|-------------|
| `consensusAddr` | Block signing address (`geth account new`) |
| `feeAddr` | Legacy BC fee address (use same as consensusAddr for private chains) |
| `bscFeeAddr` | Address that receives block rewards |
| `votingPower` | Voting weight, use `0x64` (= 100) for equal weight |
| `bLSPublicKey` | BLS12-381 public key for Fast Finality (`geth bls account new`, 96 hex chars) |

`validators.conf` is gitignored and must be created locally.

### Step 3: Configure chain parameters

Chain parameters are defined at the top of the `Makefile` with sensible private-chain defaults.
Edit them directly, or override on the command line:

```bash
make build CHAIN_ID=1234 MAX_ELECTED_VALIDATORS=5

# Values containing spaces must be quoted:
make build BLOCK_INTERVAL="5 seconds" BREATHE_BLOCK_INTERVAL="30 minutes"
```

All parameters:

| Variable | Default | Description |
|----------|---------|-------------|
| `CHAIN_ID` | `714` | Chain ID (`bscChainID` in System.sol) |
| `MAX_ELECTED_VALIDATORS` | `3` | Max active validators (StakeHub) |
| `BLOCK_INTERVAL` | `3 seconds` | Block time, used to convert time params to block counts (BSCGovernor) |
| `BREATHE_BLOCK_INTERVAL` | `10 minutes` | Interval for StakeHub reward distribution and validator state updates |
| `UNBOND_PERIOD` | `2 minutes` | Wait time after undelegating before funds can be withdrawn (StakeHub) |
| `DOWNTIME_JAIL_TIME` | `2 minutes` | Jail duration after misdemeanor (StakeHub) |
| `FELONY_JAIL_TIME` | `3 minutes` | Jail duration after felony (StakeHub) |
| `MISDEMEANOR_THRESHOLD` | `50` | Slash count before misdemeanor penalty (SlashIndicator) |
| `FELONY_THRESHOLD` | `150` | Slash count before felony and jail (SlashIndicator) |
| `FELONY_SLASH_SCOPE` | `60` | Evidence validity window in blocks for double-sign/malicious vote (SlashIndicator) |
| `INIT_BURN_RATIO` | `1000` | Block fee burn ratio out of 10000, i.e. 10% (BSCValidatorSet) |
| `STAKE_HUB_PROTECTOR` | `address(0xdEaD)` | Address that can emergency-pause StakeHub; `0xdEaD` disables this |
| `GOVERNOR_PROTECTOR` | `address(0xdEaD)` | Address that can emergency-pause BSCGovernor |
| `TOKEN_RECOVER_PORTAL_PROTECTOR` | `address(0xdEaD)` | Address that can emergency-pause TokenRecoverPortal |
| `SOURCE_CHAIN_ID` | `Binance-Chain-Ganges` | Legacy BC chain name used in token recovery signatures; irrelevant for private chains |
| `INIT_VOTING_PERIOD` | `2 minutes / BLOCK_INTERVAL` | Governance voting duration in blocks (BSCGovernor) |
| `INIT_MIN_PERIOD_AFTER_QUORUM` | `uint64(1 minutes / BLOCK_INTERVAL)` | Minimum voting time after quorum is reached (BSCGovernor) |
| `INIT_MINIMAL_DELAY` | `1 minutes` | Timelock delay between proposal queuing and execution (BSCTimelock) |

### Step 4: Build

```bash
make build
```

This runs the full pipeline:
1. Copies `validators.conf` into `bsc-genesis-contract/` and generates `validators.js`
2. Modifies contract source files with the configured chain parameters
3. Compiles all contracts via `forge build`
4. Restores contract source files to original state
5. Extracts `.deployedBytecode.object` from each artifact and writes it to the corresponding file here

> If the build is interrupted before step 4, run `make recover` to restore contract source files.

### Step 5: Rebuild the Go binary

```bash
cd /path/to/abcore
make geth
```

The bytecode files are embedded at compile time via `//go:embed`, so any change requires a Go rebuild.

## Notes

- `deployedBytecode` (not `bytecode`) must be used. `bytecode` includes the constructor, which is
  not present in an already-deployed contract. `SetCode` in the EVM writes runtime bytecode directly.
- All contracts have their nonce set to 1 after injection, as done in `applyParliaGenesisUpgrade`
  (`core/systemcontracts/upgrade.go`).
- The bytecode is deterministic for a given set of parameters and compiler version. Changing any
  chain parameter requires regenerating the bytecodes and rebuilding the Go binary.
